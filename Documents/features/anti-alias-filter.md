# ドット絵エディタ用 静的MLAA（形態学的アンチエイリアス）実装仕様書

## 1. プロジェクト要件と目的
本仕様書は、ドット絵エディタにおける「静的な画像フィルタ」として、対象画像の線の太さ（シルエット）を物理的・視覚的に変化させることなく、ジャギー（階段状の境界）のみを平滑化するアンチエイリアスアルゴリズムの実装要件を定義する。

### 1.1. アルゴリズム選定理由（MLAAアプローチ）

* **MLAAの優位性:** 形態学的アンチエイリアス（MLAA）は、画像の色の不連続性（エッジ）を検出し、その形状パターン（L字、Z字、U字など）を分類して、該当するピクセルの「カバレッジ（面積）」を計算する手法である。
* **効果:** 直線エッジや単純な45度線には干渉せず、階段状の「角（コーナー）」のみに局所的なアルファブレンドを適用するため、線の太さを維持したままジャギーを解消できる。

---

## 2. アルゴリズム・パイプライン
実装は論理的に以下の3フェーズ（Pass）に分割して構築すること。

### Pass 1: エッジ検出 (Edge Detection)
画像の輝度（Luminance）または色差を評価し、ピクセル間の境界（不連続性）を検出する。

* **処理内容:** 各ピクセルについて、右隣および下隣のピクセルとの輝度差を閾値（Threshold）と比較する。
* **出力:** 境界の有無を示すビットマスク（またはブール型配列）。エッジが存在する場合はフラグを立てる。

### Pass 2: パターン分類とカバレッジ計算 (Pattern Detection & Coverage)
検出されたエッジ線の両端を探索し、ジャギーの形状パターンを特定する。

* **処理内容:**
    1. エッジに沿ってピクセルを走査し、交差するエッジ（曲がり角）を見つける。
    2. 抽出された形状（L字、Z字など）を事前定義されたルール（またはLUT）と照合する。
    3. 対象ピクセルが元の図形に占める面積（カバレッジ $a$）を算出する。
* **ドット絵向け制約:** 探索距離（Search Distance）の上限を厳格に設定する。広範囲の探索は意図せぬ長距離グラデーション（線のぼやけ）を引き起こすため、最大でも2〜3ピクセルの局所探索に留める。

### Pass 3: ブレンド合成 (Final Blending)
Pass 2で得られたカバレッジ（ブレンド重み）に基づき、最終的な色を合成する。

* **処理内容:** カバレッジの比率に応じて、対象ピクセルの色と隣接するエッジ外のピクセルの色を線形補間（Lerp）する。
* **ドット絵向け制約（非対称ブレンド）:**
  明度の高い色（背景）が暗い色（線画）を侵食すると線が細く見え、逆の場合は太く見える錯覚を防ぐため、以下のロジックを組み込む。
  * 元ピクセルの輝度が隣接ピクセルより低い（暗い＝主線である）場合: ブレンド係数を抑制し、元の暗い色を強く残す。
  * 係数のバイアス例: `重み = 暗い側 ? カバレッジ * 0.7 : カバレッジ * 1.3` (クランプ必須)。

## 3. C# 実装の技術的制約 (AIエージェント向け指示)

| 項目 | 制約事項・指示 |
| :--- | :--- |
| **メモリアクセス** | `Bitmap.GetPixel` / `SetPixel` の使用は**厳禁**とする。非常に低速であるため。 |
| **ポインタ操作** | 処理対象画像（Bitmapまたは同等のバッファ）へのアクセスは、必ず `Bitmap.LockBits` を用いたメモリロックと、`unsafe` ブロック内でのポインタ演算（`byte*` や `int*`）によって実装すること。 |
| **Strideの考慮** | ポインタ演算時は、画像の幅だけでなく必ず `BitmapData.Stride`（行のパディングを含む実際のバイト幅）を計算に含めること。 |
| **並列化** | 完全な静的画像フィルタリングであるため、エッジ検出やブレンド処理の各行単位のループには `Parallel.For` などのマルチスレッド処理を導入し、パフォーマンスを最大化すること。 |
| **アルファチャンネル** | 透明度（Alpha）を含む画像（ARGB32）に対応すること。アルファ値が0（完全透明）のピクセルはエッジ判定から除外、または輝度計算において黒（背景）と同等に扱うなどの例外処理を明記すること。 |

## 5. UI/UX 要件

### 5.1. 設置場所
* **配置場所:** `PictureActionMenu`（右側の垂直ツールバー）内、**拡大縮小ボタンの直下**に配置する。
* **アイコン:** アンチエイリアスの効果（斜め線の平滑化）を想起させるアイコン（例：魔法の杖、または専用の平滑化アイコン）。

### 5.2. インタラクション
* **単発実行:** ボタン押下により、現在編集中のレイヤー全体、または選択範囲に対して即座にフィルタを適用する。
* **Undo/Redo 対応:** フィルタ適用は破壊的な変更となるため、必ずコマンド履歴に登録し、`Ctrl+Z` で取り消し可能にすること。（プレビュー機能は Undo により代替されるため、本バージョンでは実装不要とする。）

## 6. ユーザーストーリー

- 作業エリア、または選択範囲に対して、右側ツールバーのアンチエイリアスボタンを押すことで、形状を維持したままジャギーを平滑化したい。
- 実行後、結果が意図に沿わない場合は `Undo` ですぐに元の状態に戻したい。
