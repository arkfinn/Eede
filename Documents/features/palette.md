# パレット管理機能

## 1. 基本コンセプト
- [x] 複数のパレットをタブ形式で管理し、用途に応じて切り替え可能とする。
- [x] 「ファイルベースのパレット」と「一時パレット」を区別して管理する。

## 2. 機能要件
### タブ管理 (新規実装)
- [x] ファイルから開いたパレットは新しいタブで開く。
- [x] 同じファイルを再度開こうとした場合は、新しくタブを作らず、既存の対象タブをアクティブ（フォーカス）にする。
- [x] 新しいタブの名称はファイル名（拡張子を除く）とする。（例: `test.aact` → `test`）
- [x] ファイルから開いたパレットのタブは個別に閉じることができる。
- [ ] タブの順序はドラッグ＆ドロップで入れ替え可能とする。

### 保存フロー (既存ロジックの拡張)
- [x] **Dirty（変更）管理 (新規実装)**:
    - [x] パレット編集時に「変更あり」の状態を管理する。
    - [x] タブを閉じる際、Dirty状態であれば保存するか破棄するかを確認する。
    - [x] 一時パレットは既存通りDirty管理を行わない（閉じても確認しない）。
- [x] **上書き保存 (新規実装)**:
    - [x] ファイルと紐付いているパレットは、ショートカット（Ctrl+S等）またはメニューから直接上書き保存を可能とする。
- [x] **名前を付けて保存 / 一時パレットの保存 (既存ロジック流用)**:
    - [x] 既存の `SavePaletteCommand`（ファイルピッカーを介した保存）のロジックを再利用し、各タブで実行可能とする。

### 読み込み (既存ロジック流用)
- [x] 既存の `LoadPaletteCommand`（ファイルピッカーを介した読み込み）および `PaletteRepository` によるパース処理を再利用する。
- [x] 読み込み先を「現在のパレットの置換」から「新しいタブの追加」に変更する。
- [x] アプリケーションへのファイルのドラッグ＆ドロップによるパレットの読み込みに対応する。

## 3. UI/UX要件
- [x] **タブUIの導入 (新規実装)**:
    - [x] `PaletteContainer.axaml` に `TabControl` を導入し、各タブ内で既存の `PaletteForm` を再利用する。
- [x] **Dirty状態の視覚化 (新規実装)**:
    - [x] 変更があるタブには、名称の末尾に `*` を表示するなどのフィードバックを行う。
- [x] **タブのコンテキストメニュー (新規実装)**:
    - [x] 右クリックから「保存」「閉じる」「これ以外をすべて閉じる」を可能とする。
- [x] **デザイン**:
    - [x] `DESIGN.md` に従い、角丸0pxのハードエッジなタブデザインを採用する。
- [ ] **タブの溢れ対応**:
    - [ ] タブが多くなった場合は、スクロールまたはドロップダウンによる選択を可能とする。

## 4. 永続化（再起動時の復元） (新規実装)
- [x] アプリケーション終了時に開いていたパレットの状態（ファイルパス、タブの並び順、アクティブだったタブ）を保存する。
- [x] 次回起動時にこれらを自動的に復元する。
- [x] 復元対象のファイルが削除・移動されていた場合は、エラーを表示せず、そのタブの復元をスキップする。

## 5. 例外系・エッジケース
- [ ] **外部変更検知 (新規実装)**:
    - [ ] 開いているパレットファイルが外部で更新された場合、再読み込みの確認を行う。
- [ ] **保存失敗**:
    - [ ] 書き込み権限不足などのエラーハンドリングを行う。
