# 軌跡仕様書: ドックエリアの拡大縮小機能

## 1. 概要
ドックエリア（右側の画像管理領域）の各タブにおいて、表示されている画像の表示倍率を個別に変更できる機能を導入する。これにより、小さなドット絵の細部確認や、大きな画像の全体像の把握が容易になる。

## 2. 機能要件
- **タブごとの独立した倍率管理:**
  - 各ドックタブは自身の表示倍率を保持し、タブ切り替え時にその倍率が維持されること。
- **拡大縮小操作:**
  - 各タブのUI（ヘッダーまたはフッター付近）に、作業エリアと同様の拡大縮小ボタン（+ / -）を配置する。
  - 作業エリアで使用されているものと同じ倍率ステップ（例: 100%, 200%, 400%, 800%...）を採用する。
- **初期表示状態:**
  - 新しく画像がドックに追加された際、またはタブが開かれた際の初期倍率は「100%（原寸）」とする。
- **描画内容:**
  - 拡大時には、作業エリアの画像と同様の品質（ニアレストネイバー等）で描画されること。

## 3. 非機能要件
- **パフォーマンス:**
  - 拡大表示中もドックエリアのスクロールやタブ切り替えがスムーズに行えること。
- **一貫性:**
  - ボタンのデザインやスケーリングの挙動は、作業エリアのUIと統一感を持たせる。

## 4. 受け入れ基準
- [ ] ドックの各タブに拡大縮小用のボタンが表示されている。
- [ ] ボタンを押すことで画像の表示サイズが変化する。
- [ ] 拡大倍率が作業エリアのステップと一致している。
- [ ] タブAを400%、タブBを100%にした状態で切り替えた際、それぞれの倍率が保持されている。
- [ ] 新規読み込み時の倍率が100%であること。

## 5. 範囲外 (Out of Scope)
- マウスホイールによる拡大縮小（今回はボタン操作を優先）。
- ドックエリア全体のUI要素（テキストやタブ自体のサイズ）のスケーリング。
- 「ウィンドウに合わせる（Fit to View）」機能の自動発動。
