# Track: ツールバーでのグリッドサイズ設定の永続化

## 1. Overview (概要)
ユーザーがツールバーで設定したグリッドサイズ（幅・高さ）をアプリケーション終了後も保持し、次回起動時に自動的に復元する機能を提供します。これにより、ユーザーは起動のたびに好みのグリッドサイズを再設定する手間を省くことができます。

## 2. Functional Requirements (機能要件)
- **設定の保存**: ツールバーのグリッドサイズ（幅・高さ）が変更された際、即座にローカル設定ファイルへ保存する。
- **設定の復元**: アプリケーション起動時に設定ファイルを読み込み、前回のグリッドサイズをツールバーの初期値として反映する。
- **保存場所**: ユーザーのローカルアプリケーションデータフォルダ（%AppData%\Eede\settings.json）に保存する。
- **保存形式**: JSON 形式。
- **デフォルト値**: 設定ファイルが存在しない、または読み込みに失敗した場合は、現在のデフォルト値（32x32）を使用する。

## 3. Non-Functional Requirements (非機能要件)
- **堅牢性**: 設定ファイルの読み書きに失敗しても、アプリケーションがクラッシュせず、デフォルト値で正常に起動すること。
- **パフォーマンス**: 保存処理は軽量な JSON 書き出しとし、ユーザー操作を妨げないこと。

## 4. Acceptance Criteria (受け入れ条件)
- [ ] ツールバーでグリッドサイズの幅・高さを変更し、アプリを再起動した際、変更後のサイズが選択されていること。
- [ ] 設定ファイル（\settings.json\）が正しいパスに作成され、変更が反映されていること。
- [ ] 設定ファイルを手動で削除または破損させた状態で起動しても、デフォルトの32x32で正常に起動すること。

## 5. Out of Scope (対象外)
- グリッドの表示・非表示状態の保存（今回は「サイズ」のみ）。
- キャンバスごとの個別設定（アプリ全体で一つのデフォルト値を共有する）。
- その他の設定項目（テーマ、背景色など）の永続化（将来の拡張に含める）。
