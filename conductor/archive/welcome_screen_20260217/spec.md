# Track Specification: Welcome Screen Enhancement

## 1. Overview
ドックエリアに存在する「ようこそ」画面の中身を、実用的かつモダンなデザインに刷新する。
ロゴを配した洗練されたレイアウトにより、ブランドイメージを高めつつ、スムーズな作業開始を支援する。

## 2. Functional Requirements
- **ロゴ表示:** アプリケーションのロゴを上部に表示する。
- **最近編集した画像リンク:** 直近で使用したファイルのリストを表示し、クリックでその画像を開く。
- **クイックアクション:** 「新しく画像を作る」「画像を開く」ためのショートカットボタンを配置する。
- **外部リンク:** 
    - GitHub リポジトリへのリンク
    - GitHub Issues (儀路板) へのリンク
- **アプリ情報:** 現在のバージョン情報を表示する。
- **レイアウト:** ドックエリアの狭い幅でも視認性を保ちつつ、スクロール可能な構造とする。

## 3. Non-Functional Requirements
- **デザイン:** ダークテーマに最適化された、フラットでモダンな「エンジニアツール風」のインターフェース。ロゴをアクセントとして配置。
- **保守性:** 既存の `DialogButtons` や `PathIcon`（SVGリソース）などの共通部品を活用し、一貫性を保つ。

## 4. Acceptance Criteria
- [ ] ドックの「ようこそ」タブの上部にロゴが表示されていること。
- [ ] ドックの「ようこそ」タブを開いた際、新しいデザインが表示されること。
- [ ] 「新規作成」「開く」ボタンが正しく動作すること。
- [ ] GitHub へのリンクをクリックするとブラウザで開くこと。
- [ ] 最近編集した画像が保存・表示され、クリックで開けること。
- [ ] ドックの幅を狭めても、文字が重なったりせず、適切に表示されること。

## 5. Out of Scope
- 作業エリア（メインキャンバス）側への「ようこそ」表示。
