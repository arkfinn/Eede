# トラック仕様書: SVGアイコンコンポーネント基盤の構築と「透過ボタン」の移行

## 1. 概要
Eede プロジェクトで使用されているビットマップ形式（bmp, png）のアイコンを、保守性と柔軟性の高い SVG 形式へ段階的に移行するための基盤を構築する。最初の適用例として「透過ボタン（blur_on）」を SVG 化する。

## 2. 機能要件
- **SVG パスのリソース化:** 各アイコンの SVG データを StreamGeometry として定義し、1 ファイル 1 アイコンの粒度でリソースファイルを作成する。
- **中央集約管理:** 個別のアイコンファイルを MergedDictionaries を用いて Icons.axaml に集約し、アプリ全体から参照可能にする。
- **統一されたアイコン表示:** PathIcon コントロールと共通の .icon クラスを使用し、スタイルを統一する。
- **「透過ボタン」の差し替え:** 既存のビットマップアイコンを使用している箇所を、新規作成した SVG アイコンコンポーネントに差し替える。

## 3. 非機能要件
- **サイズと色の統一:** スタイルにより 16x16 サイズを強制し、ill: currentColor 相当の挙動（親の Foreground 継承）を実現する。
- **メンテナンス性:** 複雑な SVG パスがコードを圧迫しないよう、適切なディレクトリ構造（Resources/Icons/）で管理する。

## 4. 受け入れ条件
- [ ] Resources/Icons/BlurOn.axaml が作成され、指定された SVG パスが定義されていること。
- [ ] Resources/Icons.axaml を介してアプリ全体でアイコンリソースが利用可能であること。
- [ ] Styles/ 以下に .icon クラスを持つ PathIcon 用の共通スタイルが定義されていること。
- [ ] 透過ボタンの表示が新しい SVG アイコンに置き換わり、16x16 サイズかつテーマに応じた色で表示されること。

## 5. 範囲外 (Out of Scope)
- 今回指定された「透過ボタン」以外の全アイコンの一斉置換（本トラックは基盤構築と最初の 1 例を目的とする）。
